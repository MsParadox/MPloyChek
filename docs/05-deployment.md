## Production Architecture

```
Browser
  │
  ▼
Vercel (Angular SPA)
  │  API calls (HTTPS)                  WebSocket (WSS)
  ▼                                            ▼
Render (Node.js / Express)  ←──────────────────┘
  │
  ├── Neon PostgreSQL (DATABASE_URL)
  ├── Cloudinary (file storage)
  └── Brevo (email)

GitHub → GitHub Actions → Render (deploy) + Vercel (deploy)
```

## CI/CD Flow

Every push to `main`:
1. **Test** — Jest runs all backend tests
2. **Lint/Type-check** — `tsc --noEmit`
3. **Deploy backend** — Render API hook
4. **Build frontend** — `ng build --configuration production`
5. **Deploy frontend** — Vercel

Every pull request:
1. Type-check and test
2. Preview deployment on Vercel

## Important: Free Tier Notes

**Render** services sleep after 15 min of inactivity. First request after sleep ~30s.  
→ Add [UptimeRobot](https://uptimerobot.com) (free) to ping every 5 minutes.

**Neon** databases pause after 5 days of no queries.  
→ Visit neon.tech dashboard → Resume if needed.

**Render** re-runs `npx prisma migrate deploy` on every boot (idempotent — safe).
